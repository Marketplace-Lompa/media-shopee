# Visual Identity Analysis and Guidelines Synthesis

## Purpose

Adicionar ao cerebro global a capacidade de:

- analisar identidade visual de referencias
- organizar repertorio visual
- sintetizar guidelines reutilizaveis por estilo

## Role in the Global Brain

Este arquivo conecta:

- casos reais (`case-*.md`)
- taxonomia de estilos
- regras gerais de design
- direcao criativa baseada em repertorio

## Core Workflow

## 1. Collect references

- URLs reais (sites/templates)
- screenshots de casos
- referências externas curadas (ex.: [Behance UI/UX](https://www.behance.net/galleries/ui-ux))

## 2. Extract identity signals

- tipografia
- contraste
- composicao
- superficies
- motion
- pointer/cursor signature
- tom de marca (corporativo / playful / editorial / disruptivo)

## 3. Classify style

Usar `taxonomias/style-levels.md` para classificar os eixos.

## 4. Synthesize guidelines

Transformar observacao em:

- principios
- tokens
- component rules
- responsive guardrails
- anti-padroes

## 5. Promote to reusable knowledge

Se repetir em mais de um caso:

- mover para `design-rules-and-heuristics.md`
- mover para `component-patterns.md`
- atualizar indices por estilo

## Minimum Deliverable for Identity Analysis

1. Assinatura visual dominante
2. Classificacao por eixos
3. Guidelines sintetizadas
4. Guardrails mobile/tablet/desktop
5. Anti-padroes
6. O que e referencia vs o que vira direcao original

## Behance Usage Policy (Reference Only)

Fonte de repertorio recomendada:

- [Behance UI/UX Galleries](https://www.behance.net/galleries/ui-ux)

Uso correto:

- ampliar repertorio visual
- comparar linguagens visuais
- identificar tendências de composição, tipografia e interfaces
- converter em diretrizes abstratas

Uso incorreto:

- copiar peças específicas 1:1
- tratar uma peça de portfólio como especificação de produção sem adaptação
- replicar identidade alheia sem contextualização

## Recommended Reference Platforms (Padrao Ouro)

Para ampliar repertorio com mais confiabilidade e diversidade de sinais, usar o catalogo:

- `design-reference-platforms-catalog.md`

Composicao recomendada de fontes:

- `produto real / UX flows` (ex.: Mobbin, Page Flows)
- `website inspiration` (ex.: Lapa, One Page Love, Awwwards)
- `repertorio visual amplo` (ex.: Behance)
- `design system oficial` (ex.: Material, HIG, Carbon, Polaris)

## Identity Analysis Checklist (Quick)

- O CTA continua dominante apesar da direção visual?
- A tipografia lidera ou compete com ornamentação?
- A composição é escalável para mobile?
- Os elementos de assinatura (mockup/collage/cursor) têm fallback/responsive strategy?
- O sistema visual pode ser descrito em tokens/regras?

## Behance UI/UX Gallery Deep-Dive (2026-02-25)

Estudo aplicado e promovido para o cerebro global:

- `visual-identity-behance-uiux-trend-scan.md`

Principais aprendizados promovidos:

- A curadoria de UI/UX mistura fortemente `website + branding + product UI`, nao apenas telas de produto.
- Clusters uteis para classificacao rapida:
  - `brand_product_hybrid`
  - `proof_led_saas`
  - `dark_fintech_premium`
  - `editorial_alternativo`
  - `system_component_rigor`
- Em direcoes autorais/editoriais, mobile exige **reposicionamento** de overlays (nao so scale-down).
- Behance funciona melhor como repertorio de **sinais visuais** e **guidelines abstratas**, nao como blueprint.
